ConfigMaps

放一些不私密的键值对配置. pod可通过环境变量, 命令行参数, 或者卷中的配置文件取得ConfigMap.

配置将自动但可能不及时地更新.

apiVersion: v1
kind: ConfigMap
metadata:
  name: game-demo
data:
  # property-like keys; each key maps to a simple value
  player_initial_lives: "3"
  ui_properties_file_name: "user-interface.properties"
  #
  # file-like keys
  game.properties: |
    enemy.types=aliens,monsters
    player.maximum-lives=5    
  user-interface.properties: |
    color.good=purple
    color.bad=yellow
    allow.textmode=true

使用

pod的配置中可以引用同namespace的ConfigMap, 引用方法有:

  1. entrypoint的命令行参数
  2. 环境变量
  3. 放到文件系统作为一个只读卷
  4. 写一些代码访问kubernetes提供的参数访问API

apiVersion: v1
kind: Pod
metadata:
  name: configmap-demo-pod
spec:
  containers:
    - name: demo
      image: game.example/demo-game
      env: # 环境变量使用示例
        # Define the environment variable
        - name: PLAYER_INITIAL_LIVES # Notice that the case is different here
                                     # from the key name in the ConfigMap.
          valueFrom:
            configMapKeyRef:
              name: game-demo           # The ConfigMap this value comes from.
              key: player_initial_lives # The key to fetch.
        - name: UI_PROPERTIES_FILE_NAME
          valueFrom:
            configMapKeyRef:
              name: game-demo
              key: ui_properties_file_name
      volumeMounts: # 挂载示例, 该卷由.spec.volumes[*]定义
      - name: config
        mountPath: "/config"
        readOnly: true
  volumes:
    # You set volumes at the Pod level, then mount them into containers inside that Pod
    - name: config
      configMap:
        # Provide the name of the ConfigMap you want to mount.
        name: game-demo
        # An array of keys from the ConfigMap to create as files
        # 如果这一项不写, 则所有在该ConfigMap的文件全部会在/config中出现
        items:
        - key: "game.properties" # 上述定义的键
          path: "game.properties" # mount到的相对路径
        - key: "user-interface.properties"
          path: "user-interface.properties"

Secret

放敏感信息, 例如密码, ssh密钥, OAuth token.

定义

通过文件创建:

echo -n 'admin' > ./username.txt
echo -n '1f2d1e2e67df' > ./password.txt
kubectl create secret generic db-user-pass \
  --from-file=./username.txt \
  --from-file=./password.txt \
  --from-literal=password2='S!B\*d$zDsb='

  name: mysecret

  username: YWRtaW4= 
  password: MWYyZDFlMmU2N2Rm
# 或者使用

  config.yaml: |-
    apiUrl: "https://my.api.com/api/v1"
    username: username
    password: password

pod可以引用secret:

  • 挂载为文件
  • 环境变量
  • 拉镜像时kubelet使用

apiVersion: v1
kind: Pod
metadata:
  name: mypod
spec:
  containers:
  - name: test-container
    image: busybox
    command: ["/bin/sh", "-c", "env"]
    envFrom: # envFrom将引入该secret的所有变量
    - secretRef:
        name: mysecret
  - name: mypod
    image: redis
    env: # 环境变量使用示例
      - name: SECRET_USERNAME
        valueFrom:
          secretKeyRef:
            name: mysecret
            key: username
    volumeMounts: # 文件系统使用示例
    - name: foo
      mountPath: "/etc/foo" # 所有文件将以"文件名/文件内容"的形式表示"键/值"
      readOnly: true
  volumes:
  - name: foo
    secret:
      secretName: mysecret
      items:
      - key: username
        path: my-group/my-username # 改变默认的路径, 默认路径同键名

容器资源管理

pod的spec.containers[].resources一项, 包含requests与limits两种选项.

requests选项如果初始化不满足, 拒绝初始化.

limits选项若是内存超出, 容器将被终止并重启, 若是CPU超出, 并不会重启.

可以选择的资源有:

  • cpu: 一个单位CPU定义为1 vCPU/Core或1 hyperthread. 可以不为整数, 比如1, 1.0, 1500m(=1.5)等.
  • memory: 可选单位包括十进值单位K, M, G, T… 二进制单位Ki, Mi…
